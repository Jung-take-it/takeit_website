13장. 비동기 프로그래밍, Promise, Fetch, Async/Await

1. 비동기 프로그래밍이란?

비동기 프로그래밍은 시간이 걸리는 작업(예: 서버 요청)을 기다리지 않고, 다른 작업을 먼저 수행하는 방식입니다.

2. Promise

Promise는 미래의 완료(또는 실패) 여부에 따라 결과를 처리하는 객체입니다.

const myPromise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("성공!"), 1000);
});

myPromise.then(result => console.log(result)).catch(err => console.error(err));

3. Fetch API

fetch는 네트워크 요청을 보내고, 그 결과를 Promise로 반환합니다.

fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(res => res.json())
  .then(data => console.log(data));

4. async / await

Promise 기반 비동기 작업을 더 쉽게 작성할 수 있는 문법입니다.

async function getData() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("에러 발생:", error);
  }
}
getData();
